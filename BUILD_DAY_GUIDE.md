# Claude Build Day — Participant Guide (reference)

> Stored verbatim-condensed from the official participant guide for the team. Source of truth for rules, scoring, schedule, submission.

## Goal
Build something new (a working app from a standing start) or point Claude at your product's biggest problem. Leverage dynamic workflows / orchestration; let Claude carry the work. $500 in API credits (24h expiry) provided.
1. Start with a clear **brief** (problem, who it's for, what "done" looks like).
2. Brief Claude (repo, answer its questions, set goal + rubric), then let it go.
3. **Ship**: deploy to a live URL; submit with brief, rubric, and session log.

## Scoring
- **Impact 35%** — real-world potential; who benefits, how much; could people use it; fits a problem statement.
- **Demo 35%** — working, impressive, holds up live, proves the impact.
- **Opus 4.8 Use 15%** — creative use beyond basic integration; surface surprising capabilities.
- **Orchestration 15%** — judged from brief + rubric + workflow scripts. Is "done" verifiable by the model without a human (test suite, responding URL, rubric file it grades against)? Simple, repeatable, rerunnable tomorrow on a new problem.

## Rules
- **Repo MUST be public** (open source).
- Team **max 4** (solo allowed).
- Demo must highlight **only what your team built during the event** — judges must clearly identify your original contributions. Failure = **immediate disqualification**.
- May bring prior projects to augment, BUT the submission must be a **standalone public GitHub repo containing all code shown in the demo** (extractable into its own repo).
- **DQ** if: violates legal/ethical/platform policy, or uses code/data/assets you don't have rights to.

## Non-competitive / banned projects
AI Mental Health Advisor · Basic RAG apps · Streamlit apps · Image Analyzers · "AI for Education" chatbot · AI Job Application Screener · AI Nutrition Coach · Personality Analyzers · Medical advice bots · **any project where a dashboard is the main feature** · Sports analyzers/coaches.

## Submission (5:00 PM sharp)
Submit at the Cerebral Valley portal: a **≤1-min demo video** (only what you built), public GitHub link, all team members added. Double-check repo is public + demo link works.

## Judging
- **Round 1:** async review of video + repo → finalists.
- **Round 2:** top 6 present live, **3 min demo + 1–2 min Q&A**, equal-weighted criteria. One intro slide only; show code + product on screen, how you directed Claude, and a moment it caught/fixed its own failure.

## Prizes
1st: $100,000 API credits · 2nd: $40,000 · 3rd: $10,000.
