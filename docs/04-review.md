# Tier 4 — Review

> Review the reduction. Does the irreducible core still deliver the vision? Where does it strain?

## Does the core still solve both problems?

- **Problem 1 (emotional blindness):** YES — the incoming read replaces threat-default with grounded subtext/tone; the outgoing catch prevents wounds. ✓
- **Problem 2 (memory loss):** YES — the per-person model *is* the external relational memory; it persists what the 3-hour window can't. ✓
- **Unification holds:** one engine, two faces (emotion + memory). ✓

## Strains / open risks

- **Cold start.** Zero history on a new person = no baseline = the model could guess, and a wrong guess *feeds* paranoia (the exact failure mode). **Resolution:** on a new person Aida is visibly humble — *"still learning how they write"* — offering only clearly-labeled generic reads until it has ground. Honesty on turn one is how it earns trust. → carry to Tiers 5 & 6.
- **Calibration.** Better to say nothing than to assert subtext that isn't there. **Confidence-gating is load-bearing, not optional.**
- **The pulse is an alarm by design** — it must be the *only* alarm. Everything else stays ambient, or Aida becomes the nagging-scold anti-vision.

## Verdict

Core holds. Two items **promoted to required**: cold-start humility, and confidence-gating. Proceed to UX.
